github_码云操作---提交中的常用命令

2021年07月22日 留言

将login分支中的代码提交到本地仓库，合并到master分支推送到远端

  1. git status:查看当前项目中源代码的状态
  2. git add .:将修改后文件和新增文件添加到暂存区
  3. git commit -m "提交的文本说明信息":提交暂存区代码到本地仓库
  4. git branch:查看分支
  5. git checkout -b login:新建一个user分支，并切换到这个分支，执行后，当前处于该分支，当前所有的修改也切换到了这个分支中
  6. git push -u origin lohin:将本地的login分支推动到云端origin仓储里的login分支（远端没有此分支，该分支的第一次推送）
  7. git checkout master:切换分支到master
  8. git merge login:合并分支
  9. git push:推送
  10. git remote rm origin：远程起源已存在时先输入这个

坑

  1. 报错! [rejected] master -> master (non-fast-forward) error: failed to push some refs to 'https://xxxxxxxxxxxxxxxxxxx.git'

    解决：
    1. git pull origin master --allow-unrelated-histories 同步远程仓库
    2. 如果仓库中有一个说明文件，可能会进入vim编辑模式， esc -> : -> q! -> enter退出即可
    3. git push origin master重新提交
    4. 如果提示本地未提交，就重新add commit
